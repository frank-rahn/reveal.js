Einführung

Details: Beispiel Java<->Kotlin, Grundlegendes, Multiplatform, Entwicklungsgeschichte

Ein minimales Java-Programm


            package de.rahn.java.helloworld;

            public class Application {
              public static void main(String[] args) {
                System.out.println("Hello World!");
              }
            }

Kotlin!


            fun main(args: Array<String>): Unit {
              println("Hello World!");
            }

Kürzer


            fun main(args: Array<String>) {
              println("Hello World!")
            }

Einfacher


            fun main() {
              println("Hello World!")
            }

Minimalistisch


            fun main() = println("Hello World!")

Angemessenheit: Die Wahl der Komprimierung sollte dem Problem bzw. Aufgabe angemessen sein.

Grundlegendes zu Kotlin

  • Drastische Reduktion von Boilerplate Code
  • Erzwungene Null-Sicherheit (Nullability)
  • Freundlich zu Entwicklungswerkzeugen
  • Hervorragende Interoperabilität mit Java
    • Kotlin kann in existierenden Projekte
      schrittweise eingeführt werden
  • DSLs mit Kotlin
Beispiel von Microsoft und dem SQL-Gremium (ISO/IEC) erzählen

Multiplatform

  • Kotlin/JVM für Java und Android
  • Kotlin/JS für Javascript (Transpiler)
  • Kotlin/Native mit LLVM nativen Code erstellen
    • Android, iOS, watchOS, tvOS
    • Linux, macOS, Windows
    • WebAssembly
    • arm32, arm64, x86, x86_64, MIPS, wasm32
  • Kotlin für Data Science
Kotlin/Android hängt noch an einer älteren Java Version fest
Data Science: Jupyter Notebook, Apache Zeppelin
